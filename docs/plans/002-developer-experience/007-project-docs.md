# Project Docs

Add the docs an open source contributor expects to find.

## License

ISC license. Add `LICENSE` file at repo root.

## CONTRIBUTING.md

Short and practical:
- Prerequisites (macOS version, Swift version)
- Clone + build in 2 commands
- `make check` before submitting a PR
- Code style: "run `make fmt`"

## Issue templates

- Bug report: macOS version, steps to reproduce, expected vs actual
- Feature request: use case, proposed behavior

## PR template

Checklist: ran `make check`, described change, linked issue if applicable.
